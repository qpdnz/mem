このプロジェクト専用の「Interactive Architecture Flow Map」を作ってください。

これは、複雑なシステム構成を横方向のカラムに分けて表示し、右側のフロー項目を選択すると、中央の図の中で該当する処理経路だけが強調表示されるダッシュボードUIです。

この成果物は、Webブラウザ上で動作する可視化UIとして実装してください。
ネイティブアプリ実装は不要で、単一のHTMLファイルとして配布できる形にしてください。
ただし、表現対象は必ずこのプロジェクトの実装構造・実システムフローにしてください。

重要：
このUIは汎用テンプレートではなく、このプロジェクト専用の構成図として作ってください。
汎用SaaS例、架空Webサービス例、無関係な業務ドメイン例に置き換えないでください。
最終成果物は、ブラウザで動作する単一の index.html です。
表示内容はすべてJavaScript内のデータ配列から生成し、このプロジェクトの実際の構成要素を反映してください。

React、Vue、Svelte、Vite、Next.js、Tailwind CSS、Framer Motion、npm、Node.js、ビルド環境は使わないでください。
単一の index.html ファイルだけで完結させてください。

実装形式：
- index.html だけを出力してください
- HTML、CSS、JavaScriptをすべて1つのHTMLファイル内に含めてください
- CSSは <style> タグ内に書いてください
- JavaScriptは <script> タグ内に書いてください
- Vanilla JavaScriptのみで実装してください
- npm install 不要
- ビルド不要
- 外部API不要
- 外部ライブラリ不要
- ファイルをダブルクリックしてブラウザで開くだけで動くようにしてください
- ダミーデータではなく、このプロジェクトに沿うデータを入れてください

プロジェクト反映の要件：
- ノード、カラム、フロー、ステップ、接続線はすべてJavaScript内のデータ配列から生成してください
- HTMLに固定でベタ書きするのではなく、columns、nodes、flows、edges、steps から描画してください
- 画面上の文言は、このプロジェクトの実装に沿った命名（システム名、リソース名、処理名）を優先してください

画面全体：
- ダークテーマの横長ダッシュボード
- 背景は黒に近いネイビー
- 左〜中央に大きなシステム構成図
- 右側に固定幅のサイドバー
- 上部にタイトル、説明文、ノード種別の凡例
- 見た目は高密度だが、グリッドに整列されていて読みやすい
- 画面の横幅を最大限使用し、左右の不要な余白は極力なくしてください

中央の構成図：
- 複数の縦カラムを左から右へ並べる
- 各カラムはこのプロジェクト上の役割を表す
- カラム例：
  - 入力 / 操作
  - UI レイヤー
  - エディタ処理
  - シーン状態
  - ギズモ処理
  - 永続化
  - フィードバック

各カラムの中には複数のノードカードを縦に配置してください。
ノードはこのプロジェクトの実体に合わせてください。

ノード例：
- Scene View
- Hierarchy List
- Inspector Panel
- Status Bar
- attach_scene_click_observer
- keyboard_edit
- play_toggle
- refresh_play_ui
- refresh_statusbar
- refresh_hierarchy_rows
- refresh_row_colors
- update_inspector
- save_scene_on_edit
- ResSelectedEntity
- ResPlayState
- ResActiveScene
- DatSceneFile
- CmpSceneObjectMeta
- ResGizmoInteraction
- DatGizmoHandle
- DatGizmoAxis

右サイドバー：
右側に固定幅のサイドバーを作ってください。
上部に「FLOWS」という見出しを置き、複数のフロー項目をカードで縦に並べてください。

フロー項目例：
- シーンクリック選択フロー
- キーボード編集保存フロー
- 再生切替フロー
- ギズモ操作フロー

各フロー項目にはタイトルと短い説明文を表示してください。
クリックするとそのフローが選択状態になるようにしてください。
選択中のフロー項目は黄色〜オレンジの枠線、薄い背景、発光で強調してください。
未選択の項目は暗く表示してください。

右サイドバー下部：
「STEPS」という見出しを置き、選択中フローの手順を番号付きカードで表示してください。

接続線の仕様：
- ノード間の接続はSVG pathで描画してください
- 線は直線ではなく、滑らかなベジェ曲線にしてください
- 線の色は黄色〜オレンジ
- 線には外側のぼんやりしたグローを付けてください
- 線の途中にステップ番号の丸いバッジを表示してください

初期状態：
- 最初は1つ目のフローを選択状態にしてください
- 中央図には全ノードを表示してください
- 選択中フローに関係するノードと線だけを発光させてください
- 右下のSTEPSには、そのフローの手順を表示してください

データ構造：
実装は必ずデータ駆動にしてください。

columns:
- id
- title
- description

nodes:
- id
- columnId
- row
- title
- subtitle
- type
- description

flows:
- id
- title
- description
- highlightedNodeIds
- edges
- steps

edges:
- id
- fromNodeId
- toNodeId
- stepNumber
- label

steps:
- number
- title
- description
- fromNodeId
- toNodeId

配置の注意：
- ノードの位置は columnId と row から安定して計算してください
- 各ノードは横幅と高さを揃えてください
- カラム間には薄い縦線を入れてください
- 接続線はノードの左右端から自然につないでください
- 線が重なりすぎないように曲線にしてください
- 同じカラム内や離れたカラム間の接続にも対応してください
- 中央の図は横スクロール、または固定スケールで表示できるようにしてください
- 右サイドバーは常に見える位置に固定してください

レスポンシブ：
- デスクトップ表示を優先してください
- 横幅が足りない場合は中央の図を横スクロール可能にしてください
- サイドバーは固定幅を維持してください
- ノードや線の位置が崩れないようにしてください

仕上げ：
- 開発者向けの洗練されたダークUIにしてください
- 余白、枠線、グロー、文字のコントラストを丁寧に調整してください
- 複雑でも美しく、流れが直感的に理解できる完成度にしてください

最後に必ず、完成した index.html の全文だけを出力してください。
説明文や補足は不要です。
内部事情（文字数上限、コマンド制限、apply_patch使用可否、実装手順の説明など）は一切出力しないでください。
