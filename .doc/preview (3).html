Webシステム版 v3 / visual guide

prefixごとに「ページのここだよ」を見せる命名ガイド

方向性は直近版を維持し、全prefixに対して小さい画面SVG・処理フローSVG・命名例を追加しました。 ファイル名は 連番_役割_大項目_中項目_小項目 の順です。

001連番
Ui / Val / Ope役割
table / sql / vo大項目
${domain}中項目
row / select小項目

基本形：連番_役割_大項目_中項目_小項目

320 連番_ Ui 役割_ table 大項目_ invoice 中項目/domain_ row 小項目

大項目は「ページ上の一番大きい部位」または「処理上の大きい境界」です。 中項目は order / delivery / invoice のような業務ドメイン名を入れます。

逆にしない: Ui_320_table_order_row.tsx ではなく、 320_Ui_table_order_row.tsx に統一します。

画面のここだよ：全体マップ

画面部品系の大項目は、だいたいこのWeb画面のどこにあるかで判定します。

headbar / navbar / notification
画面上部の共通操作・通知・ユーザーmenu。
sidebar / breadcrumb / tabs
現在位置と画面内移動。業務ドメインではなく、画面部位名。
search / filter / table
一覧画面の中心。小項目に input / panel / row / cell を置く。
detail / drawer / modal
1件の詳細や編集を表示する領域。
chart / map / timeline
分析・配送・履歴などの可視化領域。
error / empty / loading
成功時以外の状態表示も独立して命名する。
