汎用的な「Interactive Architecture Flow Map」を作ってください。

これは、複雑なシステム構成を横方向のカラムに分けて表示し、右側のフロー項目を選択すると、中央の図の中で該当する処理経路だけが発光して流れるダッシュボードUIです。

この成果物は、Webブラウザ上で動作する可視化UIとして実装してください。
ネイティブアプリや特定プラットフォーム専用の実装ではなく、単一のHTMLファイルとして配布できるWebページにしてください。
ただし、表現する内容は特定のWebサービスに限定せず、任意のシステム構成や業務フローに対応できる汎用テンプレートとして設計してください。

重要：
このUIは、特定の業務や特定のシステム専用ではなく、どのようなシステム構成にも適用できる汎用的なWebベースの可視化テンプレートとして作ってください。
最終成果物は、ブラウザで動作する単一の index.html です。
表示内容はすべてJavaScript内のデータ配列から生成し、データを差し替えるだけで別のシステム構成図として再利用できるようにしてください。

React、Vue、Svelte、Vite、Next.js、Tailwind CSS、Framer Motion、npm、Node.js、ビルド環境は使わないでください。
単一の index.html ファイルだけで完結させてください。

実装形式：
- index.html だけを出力してください
- HTML、CSS、JavaScriptをすべて1つのHTMLファイル内に含めてください
- CSSは <style> タグ内に書いてください
- JavaScriptは <script> タグ内に書いてください
- Vanilla JavaScriptのみで実装してください
- npm install 不要
- ビルド不要
- 外部API不要
- 外部ライブラリ不要
- ファイルをダブルクリックしてブラウザで開くだけで動くようにしてください
- ダミーデータもHTML内に含めてください

汎用性の要件：
- 特定の企業名、実在サービス名、プロダクト名、画像内に出ていた固有名詞は使わないでください
- ノード、カラム、フロー、ステップ、接続線はすべてJavaScript内のデータ配列から生成してください
- HTMLに固定でベタ書きするのではなく、columns、nodes、flows、edges、steps のデータを変更するだけで別のシステム構成図として再利用できる設計にしてください
- SaaS、社内業務システム、AIワークフロー、EC、決済、物流、医療、教育、金融、製造、分析基盤など、さまざまなシステムに対応できる構造にしてください
- 画面上に表示する文言も、特定業界に寄りすぎない汎用的なシステム用語を中心にしてください

画面全体：
- ダークテーマの横長ダッシュボード
- 背景は黒に近いネイビー
- 左〜中央に大きなシステム構成図
- 右側に固定幅のサイドバー
- 上部にタイトル、説明文、ノード種別の凡例
- UI全体は開発者向けSaaS、インフラ管理画面、ワークフロー可視化ツールのような雰囲気
- 見た目は高密度だが、グリッドに整列されていて読みやすい

中央の構成図：
- 複数の縦カラムを左から右へ並べる
- 各カラムはシステム上の役割を表す
- カラム例：
  - Users / Actors
  - Client Interfaces
  - Entry Points
  - Application Services
  - Data / Storage
  - Processing / Jobs
  - Delivery / Distribution
  - External Services

各カラムの中には複数のノードカードを縦に配置してください。
ノードは汎用的な概念名だけを使い、特定の実在サービス名、企業名、プロダクト名、画像内に出ていた固有名詞は使わないでください。

- Admin User
- External Trigger
- Web App

- Authentication Service
- API Gateway

- User Service

ノードの見た目：
- 角丸の横長カード
- 暗い背景に薄い枠線
- タイトルと短い説明文を表示
- ノード種別ごとに小さな色付きアクセントを付ける
- 通常時は暗めで控えめに表示
- 選択中フローに含まれるノードだけ、黄色〜オレンジの枠線で発光
- 関係ないノードは半透明にして背景に沈める
- 選択中ノードはわずかに明るい背景、強い枠線、外側グローを持つ
- ホバー時は少し明るくする

右サイドバー：
右側に固定幅のサイドバーを作ってください。
上部に「FLOWS」という見出しを置き、複数のフロー項目をカードで縦に並べてください。

フロー項目例：
- User Signup Flow
- Login & Session Flow
- Admin Configuration Flow
- Payment Event Flow
- File Upload & Processing Flow
- Notification Delivery Flow
- Data Export Flow
- Scheduled Background Job Flow
- Third-party Integration Flow
- Incident Monitoring Flow

各フロー項目にはタイトルと短い説明文を表示してください。
クリックするとそのフローが選択状態になるようにしてください。
選択中のフロー項目は黄色〜オレンジの枠線、薄い背景、発光で強調してください。
未選択の項目は暗く表示してください。

右サイドバー下部：
「STEPS」という見出しを置き、選択中フローの手順を番号付きカードで表示してください。

各ステップには以下を表示してください。
- 番号バッジ
- ステップタイトル
- 短い説明文

番号バッジは黄色〜オレンジの円形にしてください。
ステップ一覧は縦スクロール可能にしてください。
フローを切り替えると、ステップ一覧も自然に切り替わるようにしてください。

最も重要なインタラクション：
右側のフロー項目をクリックすると、中央のシステム構成図でそのフローの経路だけがハイライトされるようにしてください。

ハイライトの仕様：
- 選択中フローに含まれるノードだけ明るく発光
- 選択中フローに含まれる接続線だけ黄色〜オレンジで表示
- 関係ないノードと線は暗くフェード
- ただし全体構造は分かるように、非選択ノードは完全には消さない

接続線の仕様：
- ノード間の接続はSVG pathで描画してください
- 線は直線ではなく、滑らかなベジェ曲線にしてください
- 線の色は黄色〜オレンジ
- 線には外側のぼんやりしたグローを付けてください
- 線の途中にステップ番号の丸いバッジを表示してください
- バッジは線の近くに配置し、処理順が分かるようにしてください
- 矢印、粒子、グラデーションなどで進行方向を分かりやすくしてください

流れるアニメーション：
- 選択中の接続線は、血管の中を液体が流れるように連続的に動かしてください
- CSS animation、stroke-dasharray、stroke-dashoffset を使って、線そのものが流れているように見せてください
- さらに、小さな発光する粒子が SVG path に沿って始点から終点へ移動する表現を入れてください
- 粒子は1本の線につき1〜3個程度
- 粒子は一定間隔で繰り返し流れるようにしてください
- 線の上を光が脈打つように流れ、処理が進んでいる方向が直感的に分かるようにしてください
- フローを切り替えると、古い経路はフェードアウトし、新しい経路はフェードインしながら発光するようにしてください
- ノードも同時に、暗い状態から発光状態へ滑らかに切り替えてください

アニメーションの雰囲気：
- 激しすぎない
- 高級感がある
- 開発者向けプロダクトUIとして自然
- 流体、血流、電流、データパケットのような印象
- 黄色い光が経路上を順番に巡っていく
- 経路の順序と方向が一目で分かる

初期状態：
- 最初は1つ目のフローを選択状態にしてください
- 中央図には全ノードを表示してください
- 選択中フローに関係するノードと線だけを発光させてください
- 右下のSTEPSには、そのフローの手順を表示してください

データ構造：
実装は必ずデータ駆動にしてください。
特定のシステムに依存しないよう、JavaScript内に以下のような配列を定義してください。

columns:

- title
- description

nodes:
- id
- columnId
- row
- title
- subtitle
- type
- description

flows:
- id
- title
- description
- highlightedNodeIds
- edges
- steps

edges:
- id
- fromNodeId
- toNodeId
- stepNumber
- label

steps:
- number
- title
- description
- fromNodeId
- toNodeId

配置の注意：
- ノードの位置は columnId と row から安定して計算してください
- 各ノードは横幅と高さを揃えてください
- カラム間には薄い縦線を入れてください
- 接続線はノードの左右端から自然につないでください
- 線が重なりすぎないように曲線にしてください
- 同じカラム内や離れたカラム間の接続にも対応してください
- 中央の図は横スクロール、または固定スケールで表示できるようにしてください
- 右サイドバーは常に見える位置に固定してください

レスポンシブ：
- デスクトップ表示を優先してください
- 横幅が足りない場合は中央の図を横スクロール可能にしてください
- サイドバーは固定幅を維持してください
- ノードや線の位置が崩れないようにしてください

- 複雑でも美しく、流れが直感的に理解できる完成度にしてください
- データを差し替えれば、SaaS、社内業務システム、AIワークフロー、EC、決済、物流、医療、教育、金融、製造、分析基盤などに転用できるようにしてください
